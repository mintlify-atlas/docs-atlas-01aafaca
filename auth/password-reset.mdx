## Overview

The password reset flow consists of two steps:

1. **Request a token** — the API emails a reset link to the user.
2. **Reset the password** — the user submits the token and new password to complete the reset.

<Warning>
  Reset tokens expire **15 minutes** after they are issued. If the token expires, the user must request a new one by submitting their email again.
</Warning>

## Flow

<Steps>
  <Step title="Request a reset token">
    Send a `POST` request to `/api/auth/forgot-password` with the user's email address.

    The API looks up the email in the database. If a matching account is found, it:

    - Generates a random reset token.
    - Hashes the token with SHA-256 and stores the hash with a 15-minute expiry.
    - Sends an email to the user containing a link in the form:

    ```
    {FRONTEND_URL}/recuperar-password?token=<raw_token>&email=<email>
    ```

    The frontend is responsible for extracting `token` and `email` from the URL query string and passing them to step 2.

  </Step>
  <Step title="Reset the password">
    Send a `POST` request to `/api/auth/reset-password` with the token, email, and new password.

    The API:

    - Validates that all required fields are present.
    - Validates the new password against the [password rules](#password-rules).
    - Hashes the submitted token with SHA-256 and looks up the matching user record.
    - Checks that the token has not expired.
    - Hashes the new password with bcrypt (`$2y$` format) and saves it.
    - Clears the reset token and expiry from the database.

  </Step>
</Steps>

## Step 1: Request a reset token

**`POST /api/auth/forgot-password`**

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | The email address associated with the account. |

### Example request

```bash curl
curl -X POST http://localhost:4000/api/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{"email": "doctor@clinica.com"}'
```

### Responses

**`200 OK`** — Email found; reset email sent.

```json
{
  "success": true,
  "message": "Se ha enviado un correo con las instrucciones para restablecer tu contraseña"
}
```

**`404 Not Found`** — No account with that email exists.

```json
{
  "error": "No existe una cuenta con este correo"
}
```

**`500 Internal Server Error`** — The reset email could not be sent.

```json
{
  "error": "Error al enviar el correo. Por favor, intenta de nuevo."
}
```

## Step 2: Reset the password

**`POST /api/auth/reset-password`**

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | string | Yes | The raw token from the reset link query string. |
| `email` | string | Yes | The email address from the reset link query string. |
| `newPassword` | string | Yes\* | The new password. Also accepted as `password`. |

\* Either `newPassword` or `password` may be used; both refer to the same field.

### Example request

```bash curl
curl -X POST http://localhost:4000/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "<reset-token>",
    "email": "doctor@clinica.com",
    "newPassword": "NewPass@2024"
  }'
```

### Responses

**`200 OK`** — Password reset successfully.

```json
{
  "success": true,
  "message": "Tu contraseña ha sido restablecida exitosamente. Ya puedes iniciar sesión."
}
```

**`400 Bad Request`** — Validation error. Possible error messages:

```json
{ "error": "Faltan datos requeridos (token, email, password)" }
```
```json
{ "error": "La contraseña debe tener mínimo 8 caracteres." }
```
```json
{ "error": "La contraseña debe contener al menos una letra mayúscula." }
```
```json
{ "error": "La contraseña debe contener al menos un carácter especial (ej. @, #, $, !)." }
```
```json
{ "error": "La contraseña no puede contener secuencias numéricas como 123." }
```
```json
{ "error": "El token es inválido o el correo no coincide" }
```
```json
{ "error": "El token de restablecimiento ha expirado" }
```

**`500 Internal Server Error`** — Server-side error during reset.

```json
{
  "error": "Error del servidor al restablecer contraseña."
}
```

## Password rules

Every new password submitted to `/api/auth/reset-password` must satisfy all of the following:

| Rule | Details |
|------|---------|
| Minimum length | At least 8 characters |
| Uppercase letter | At least one character in the range `A`–`Z` |
| Special character | At least one non-alphanumeric character (`@`, `#`, `$`, `!`, `_`, etc.) |
| Forbidden sequence | Must not contain the substring `123` |

Passwords that fail any rule receive a `400` response with a message describing the specific violation.

---

## Notes

- **GET requests** to `/api/auth/reset-password` return `405 Method Not Allowed`. Only POST is supported.
- The `FRONTEND_URL` environment variable controls the base URL used in the reset email. It defaults to `https://dentalconnect.cloud`.
- After a successful password reset, the stored token and expiry are cleared. The same token cannot be reused.
