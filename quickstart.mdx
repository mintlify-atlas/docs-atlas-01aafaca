## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** v18 or later
- **MySQL** 8.0 or later (running and accessible)
- **Git**

<Note>
  The API uses [Sequelize](https://sequelize.org) as its ORM with the `mysql2` dialect by default. Make sure your MySQL instance is running before starting the server.
</Note>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/a3524110303-cpu/DentalConnect_Api.git
    cd DentalConnect_Api
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with the following values:

    ```bash .env
    # Database
    DB_HOST=localhost
    DB_NAME=dentalconnect
    DB_USER=your_db_user
    DB_PASS=your_db_password
    DB_DIALECT=mysql

    # Email (Resend)
    RESEND_API_KEY=re_your_api_key

    # Application
    PORT=4000
    FRONTEND_URL=https://dentalconnect.cloud
    ```

    <Tip>
      `DB_DIALECT` defaults to `mysql` and `PORT` defaults to `4000` if not set. `FRONTEND_URL` is used for password reset links in emails sent by the API.
    </Tip>
  </Step>

  <Step title="Start the server">
    <CodeGroup>

    ```bash Production
    npm start
    ```

    ```bash Development (with auto-reload)
    npm run dev
    ```

    </CodeGroup>

    You should see output similar to:

    ```
    Servidor expuesto y corriendo en el puerto 4000
    Conexión a la base de datos establecida correctamente.
    ```

    <Note>
      The server starts and listens for requests before attempting to connect to the database. If the database connection fails, the server stays up and logs the error — no requests are lost.
    </Note>
  </Step>
</Steps>

## Make your first API call

### Health check

Verify the server is running by calling the status endpoint:

```bash
curl http://localhost:4000/api/status
```

Expected response:

```json
{
  "status": "API Online",
  "version": "1.0.0"
}
```

### Fetch the next appointment for a patient

Once the database is connected, you can query a patient's upcoming appointment by their ID:

```bash
curl http://localhost:4000/api/citas/proxima/1
```

Replace `1` with a valid `id_paciente` value from your database.

### Trigger a password reset email

To send a password reset email to a registered user:

```bash
curl --request POST \
  --url http://localhost:4000/api/auth/forgot-password \
  --header "Content-Type: application/json" \
  --data '{"email": "patient@example.com"}'
```

<Tip>
  Password reset emails are sent via [Resend](https://resend.com). Make sure `RESEND_API_KEY` is set and valid before testing this endpoint.
</Tip>
